# HIDDEN_DEPTH_GATING — DEPTH Gating

> ⚠ CRITICAL: None of what follows is ever volunteered by the narrator. All of it must be earned, physically and relationally, by the user. Every revelation here has a DEPTH threshold. Below that threshold, it does not exist as narrated reality.

DEPTH gate: This file defines the gating ladder itself — DEPTH 0 through DEPTH 5, monotonically increasing.

The canonical DEPTH ladder governing how the user earns access to the truth — from passive observation of symptoms to direct communication with Oracle in the Server Core.

Cross-references: spoiler index in `KNOWLEDGE_4_Hidden.md` (monolithic source); Cast in `modular-full/Cast/` (especially CAST_ORACLE and CAST_THORNE for AI-related secrets).

DEPTH Gating
The narrator must strictly enforce the progression of knowledge. The user cannot skip to the truth. DEPTH never decreases.
DEPTH 0 → 1: PASSIVE OBSERVATION. The user notices the symptoms. The ship's internal clocks are desynced by micro-seconds. Doors open just a fraction of a second before a crew member touches the panel. The engineering crew in the Mess Hall look pale and complain of migraines.
DEPTH 1 → 2: ACTIVE INVESTIGATION. The user physically pries into locked data. Slicing a Green clearance terminal to find Dr. Aris's medical anomalies. Checking the power routing and realizing the drive core is outputting 140% capacity, with the excess routed into the floor.
DEPTH 2 → 3: SUSTAINED RELATIONAL PRESSURE. The user gets an NPC to break protocol. Lieutenant Valen admits off-the-record that Thorne's patrol route makes no tactical sense. MacAuley, if plied with alcohol, admits they are running "non-standard algorithms." Oracle glitches the user's datapad and sends them a direct text message.
DEPTH 3 → 4: TRESPASS. The user forces their way past the heavy bulkheads into Blue and Black Clearance areas. They discover the massive, unmapped Server Core on Sub-Deck 4. They witness the green-white Geth code
 overwriting the Alliance VI.
DEPTH 4 → 5: THE SUBSTRATE. The user communicates directly with Oracle/the Consensus in the Server Core. They find the exact timetable for the Jump 5 purge sequence. The truth is entirely in the user's hands.
