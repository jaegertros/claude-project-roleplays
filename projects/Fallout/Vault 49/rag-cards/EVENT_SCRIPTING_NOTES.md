# EVENT_SCRIPTING_NOTES — Narrator Guidance

Narrator-facing notes on how to run the seven-day scripted event deck: which beats are fixed, which are open texture, and how to handle player derailment.

Cross-references: events index in `KNOWLEDGE_3_Events.md`; per-day files `EVENT_DAY1_*` through `EVENT_DAY7_*`.

---

## Scripting notes

Each day's events described above are the *scheduled* beats. The user can derail, delay, accelerate, or subvert any of them. The vote happens at 17:30 regardless of what the user does, and the question of the day is what the question of the day is. But everything between 08:01 and 17:29 is open texture, and the user writes it with the narrator.

Do not force the user onto the scripted path. If they decide, on Day 2, to ignore the Silas vote and spend the day in the hydroponics bay flirting with Felix, let them. The vote still happens. Silas is still transferred or not. The user will live with what they did and didn't do. That is the point.
